# 英语盲盒互动教学系统 - 面试交付说明

## 1. 在线演示地址

- 项目页面：https://interview.zgm2003.cn/
- API 示例：https://interview.zgm2003.cn/api/boxes
- GitHub 仓库：https://github.com/zgm2003/intervirw

## 2. 项目背景

本项目来源于“中高级全栈工程师上机编程测试”。题目给定了一个英语课堂互动盲盒 HTML/CSS/原生 JavaScript Demo，原始 Demo 的核心问题是：

- 盲盒数据只存在前端内存 state 中；
- 页面刷新后状态丢失；
- 没有 RESTful API；
- 抽取状态无法由后端统一维护；
- 原 JavaScript 中存在 `boxId` 未定义的交互 bug。

本次交付目标是：保留原有 UI 和交互视觉，不重写页面结构，在最小改动下完成后端 API 搭建和前后端集成。

## 3. 技术栈

- 前端：HTML + CSS + 原生 JavaScript
- 后端：Node.js + Express
- 跨域：cors
- 测试：Node.js 内置 `node:test`
- 部署：Docker + Nginx 反向代理
- 运行端口：容器内 `3000`，服务器侧独立端口 `13000`，公网域名通过 HTTPS 访问

## 4. 已完成功能

### 4.1 后端 RESTful API

已实现以下接口：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/boxes` | 获取所有盲盒列表 |
| POST | `/api/boxes` | 创建新的盲盒 |
| PUT | `/api/boxes/:id` | 更新指定盲盒内容和图片 |
| DELETE | `/api/boxes/:id` | 删除指定盲盒 |
| POST | `/api/draw` | 随机抽取一个未抽取盲盒，并标记为已抽取 |
| POST | `/api/draw/reset` | 重置本轮抽取状态，便于重新开始 |

### 4.2 前端集成

前端已完成以下改造：

- 页面加载时调用 `GET /api/boxes` 获取数据并渲染；
- 点击“保存”时调用 `PUT /api/boxes/:id`；
- 点击“新增”时调用 `POST /api/boxes`；
- 点击“删除”时调用 `DELETE /api/boxes/:id`；
- 点击盲盒抽取时调用 `POST /api/draw`，由后端返回抽取结果；
- 修复原 Demo 中 `handleBoxClick(index)` 内部错误引用 `boxId` 的问题；
- 保留原有页面样式、动画、倒计时、模拟学生答题、前三名、点赞、结果详情等交互。

## 5. 数据与状态设计

当前按照题目要求使用服务端内存状态维护盲盒数据和抽取状态。

盲盒数据结构：

```json
{
  "id": 1,
  "content": "Apple - 苹果",
  "image": null,
  "drawn": false
}
```

说明：

- `id`：后端生成的唯一盲盒 ID；
- `content`：盲盒题目文本；
- `image`：图片 URL 或 Base64；
- `drawn`：是否已被抽取。

没有强行接入 MySQL 的原因：题目只要求“维护状态”，未要求数据库持久化。为了保证上机题可运行、可演示、低环境依赖，本版本使用内存 Store。若生产化，可以将当前 Store 平滑替换为 MySQL Repository，不影响前端 API 调用方式。

## 10. 演示路径建议

面试演示建议按以下顺序：

1. 打开 `https://interview.zgm2003.cn/`；
2. 打开浏览器 Network，刷新页面，展示 `GET /api/boxes`；

4. 新增一个盲盒，说明走 `POST /api/boxes`；

6. 点击盲盒，说明走 `POST /api/draw`，由后端保证不重复抽取；
7. 展示倒计时、模拟答题、前三名、点赞和结果页；
8. 运行或展示 `npm test` 结果。

## 11. 设计取舍

本项目没有做过度设计：

- 没有重写 UI；
- 没有引入 Vue/React；
- 没有强行接 MySQL；
- 没有设计复杂登录和权限系统；
- 没有引入 WebSocket。

原因是本题核心考察点是：

```text
能否审查现有前端状态逻辑，并用后端 RESTful API 完成最小可靠集成。
```

当前实现优先保证：

- 可运行；
- 可演示；
- API 独立可测；
- 前端原交互兼容；
- 部署入口稳定。
